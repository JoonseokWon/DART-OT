# DART-OT 이자비용 오버롤 프로세스

## 1. 실행 범위

- 사용자가 회사명, 종목코드, 고유번호, 조회기간을 입력하면 DART 정기보고서를 조회한다.
- 기본 조회기간은 최근 5개년이다.
- 대상 보고서는 사업보고서, 반기보고서, 분기보고서이다.
- 보고기간은 다음처럼 적용한다.
  - 분기보고서: 해당 분기 3개월
  - 반기보고서: 6개월
  - 사업보고서: 12개월
- 분기보고서는 누적 9개월 금액이 아니라 해당 분기 3개월의 예상 이자비용, 실제 이자비용과 매출액을 사용한다.

## 2. 보고서 원문 수집

- DART API로 회사 고유번호를 확인한다.
- 조회기간 내 정기보고서 목록을 가져온다.
- 각 보고서별로 다음 자료를 병렬 수집한다.
  - DART 재무제표 API 차입금 잔액
  - DART 재무제표 API 매출액
  - 보고서 원문 XML의 주석 문맥
- 연결 재무제표가 있으면 연결 기준을 우선한다.

## 3. 차입금/사채 주석 탐색

차입금 이자율은 보고서 원문 중 "주석" 대분류 안에서 다음 소분류를 중심으로 찾는다.

- 차입금
- 사채
- 차입금 및 사채
- 금융부채
- 전환사채, 신주인수권부사채, 교환사채

주석 제목이 정확히 잡히지 않아도, 표 안에 다음 단서가 있으면 차입금 표로 본다.

- 차입처
- 단기차입금
- 장기차입금
- 유동성장기차입금
- 사채
- 전환사채
- 이자율
- 최종만기
- 일반자금대출

## 4. 이자율 추출

차입금/사채 주석 표에서 다음 형식의 이자율을 추출한다.

- 단일 이자율: `5.10%`
- 범위 이자율: `4.72~4.89%`
- 소수 표기: `0.051`, `0.077`
- 기준금리 가산 형식: `LIBOR+0.95`, `SOFR+1.09`

처리 원칙은 다음과 같다.

- 범위 이자율은 하한과 상한의 평균을 대표 이자율로 쓴다.
- `SOFR+가산금리`는 보고기간 말 또는 그 이전의 가장 가까운 뉴욕 연준 SOFR에 공시 가산금리를 더해 참고 이자율을 계산한다.
- `LIBOR+가산금리`는 SOFR로 임의 대체하지 않는다. 계약상 대체금리와 스프레드 조정 조건을 확인할 수 없는 경우에는 추가 확인 대상으로 둔다.
- 표면이자율 0% 전환사채도 0% 이자율로 인식한다.
- 만기일 하나만 있는 경우에는 기간 시작/종료일로 보지 않고 보고기간 전체를 적용한다.
- 시작일과 종료일이 둘 이상 명시된 경우에만 실제 보유기간 일수로 가중한다.

다음 문맥은 이자율 후보에서 제외한다.

- WACC, 가중평균자본비용
- 민감도, 이자율위험, 시장위험
- 공정가치 평가, 평가손익
- 전환가액, 전환가격, 전환권, 주식수
- 차입금의존도
- 리스부채만 단독으로 표시된 행
- 이자비용 조정내역 등 설명 행

## 5. 차입금 금액 추출

차입금 잔액은 다음 우선순위로 산정한다.

1. 차입금/사채 주석의 합계 또는 총계 행
2. 차입금/사채 주석의 항목별 당기말 금액 합산
3. 재무상태표 API의 차입금 관련 계정

합계 행 예시는 다음과 같다.

- 차입금 합계
- 차입금 및 사채 합계
- 총차입금
- 차입금계
- 사채계

단위 처리 원칙은 다음과 같다.

- 주석의 `단위: 백만원`, `단위: 천원`, `단위: 원`, `단위: 억원`을 인식한다.
- 단위가 비어 있는 경우에는 숫자 크기와 주변 표를 이용해 후보 단위를 추정하되, 확정값으로 보지 않고 원문 확인 대상으로 둔다.
- 앞 표의 단위가 뒤 표에 이어져 비정상적으로 큰 금액이 만들어지는 경우에는 주변 표와 재무제표 API 금액을 비교해 후보 단위를 보정하고 원문 대조가 필요함을 남긴다.
- 같은 금액과 같은 이자율이 연결/별도 또는 중복 표에서 반복되면 한 번만 반영한다.

## 6. 실제 이자비용 추출

실제 이자비용은 주석의 금융비용 계열 표에서 찾는다.

대상 주석명은 다음을 모두 포함한다.

- 금융수익 및 금융비용
- 금융수익과 금융비용
- 금융수익 및 금융원가
- 금융원가
- 금융비용
- 재무수익 및 재무비용
- 재무수익과 재무비용
- 재무수익 및 재무원가
- 재무비용

추출 대상 계정은 다음과 같다.

- 이자비용
- 이자비용(금융원가)
- 이자비용(금융비용)
- 상각후원가 측정 금융부채 이자비용
- 차입금이자비용
- 사채이자비용
- 기타 금융부채 이자비용

처리 원칙은 다음과 같다.

- 금융수익 및 금융비용 주석 안에서 "이자비용"으로 표시된 항목은 실제 이자비용 후보로 본다.
- "이자비용" 하위에 복수 구성항목이 있고 합계 행이 있으면 합계 행을 우선한다.
- 합계 행이 없으면 차입 관련 이자비용 항목을 합산한다.
- 연결 주석과 별도 주석이 모두 있으면 연결 주석을 우선한다.
- 현금흐름표의 조정 항목인 `이자비용 조정`, `조정내역`은 실제 이자비용 후보에서 제외한다.

## 7. 적정이자율 산정

적정이자율은 단순 평균이 아니라 금액과 기간을 모두 반영한 가중평균이다.

계산식은 다음과 같다.

```text
가중평균 차입이자율 = 이자비용 적수 / 차입금 적수

이자비용 적수 = Σ(차입금 금액 × 이자율 × 보유일수)
차입금 적수 = Σ(차입금 금액 × 보유일수)
```

예시는 다음과 같다.

```text
A: 10억원, 5%, 1~12월
B: 20억원, 2%, 1~10월
C: 15억원, 7%, 1~9월
D: 10억원, 15%, 9~12월

이자비용 적수 =
10억 × 5% × 365
+ 20억 × 2% × 304
+ 15억 × 7% × 273
+ 10억 × 15% × 122

차입금 적수 =
10억 × 365
+ 20억 × 304
+ 15억 × 273
+ 10억 × 122
```

## 8. 예상 이자비용 산정

예상 이자비용은 두 방식 중 더 구체적인 방식을 사용한다.

1. 이자율과 금액이 같은 행에 있는 경우

```text
예상 이자비용 = Σ(차입금 금액 × 이자율 × 보유일수 / 365)
```

2. 이자율과 총차입금만 있는 경우

```text
예상 이자비용 = 평균차입금 × 가중평균 이자율 × 보고기간월수 / 12
```

평균차입금 산정 원칙은 다음과 같다.

- 1분기, 반기와 사업보고서는 전기말과 당기말 차입금의 평균을 사용한다.
- 3분기보고서는 해당 분기 3개월을 계산하므로 6월 말과 9월 말 차입금의 평균을 사용한다.
- 차입금 변동표에서 증가/감소가 신뢰 가능하면 변동액의 절반을 반영한다.
- 기초 잔액이 없으면 기말 잔액을 사용한다.
- 기말 잔액이 0이고 기초 잔액만 있으면 기초 잔액의 절반을 사용한다.

전환사채 관련 보정은 다음과 같다.

- 전환권조정 상각으로 보이는 행은 예상 이자비용에 가산한다.
- 연결/별도, 유동/비유동 중복 후보가 동시에 잡히면 같은 표 묶음의 대표 상각액만 사용한다.
- 전환가액, 전환권 평가, 공정가치 평가 등은 상각 후보에서 제외한다.

## 9. 중요성 프리셋과 허용차이

실행 화면에서 다음 프리셋 중 하나를 선택한다.

| 프리셋 | 전체 중요성 예시 | 오버롤 허용차이 |
| --- | --- | --- |
| 매출액 1% x 25% (기본) | 매출액 x 1% | 매출액 x 1% x 25% |
| 매출액 0.5% x 25% (보수적) | 매출액 x 0.5% | 매출액 x 0.5% x 25% |
| 세전이익 5% x 25% | 세전이익 x 5% | 세전이익 x 5% x 25% |
| 총자산 0.5% x 25% | 총자산 x 0.5% | 총자산 x 0.5% x 25% |
| 자본총계 1% x 25% | 자본총계 x 1% | 자본총계 x 1% x 25% |

감사기준은 단일 기준항목이나 백분율을 강제하지 않는다. 프리셋은 공개된 감사기준의 기준항목과 감독기관의 실무 관찰 범위를 참고해 만든 검토용 예시이며, 담당자의 전문적 판단을 대체하지 않는다.

선택한 기준금액이 DART 재무제표 API에서 검출되지 않거나 0 이하이면 다른 프리셋으로 자동 대체하지 않고 `판단불가`로 표시한다. 특히 손실 기업은 세전이익 프리셋을 사용하지 않고 매출액, 총자산 또는 자본총계가 더 적합한지 검토해야 한다.

손익 기준금액의 기간은 보고서 기간과 맞춘다.

- 분기보고서: 해당 분기 3개월 금액
- 반기보고서: 반기 누적 금액
- 사업보고서: 연간 누적 금액

총자산과 자본총계는 해당 보고기간 말 잔액을 사용한다.

## 10. 판정 로직

계산이자와 실제이자 차이를 비교한다.

```text
차이 = |예상 이자비용 - 실제 이자비용|
```

판정은 다음과 같다.

- 차이 <= 선택 프리셋의 허용차이: 기준 이내
- 차이 > 선택 프리셋의 허용차이: 추가 확인 필요
- 선택 프리셋의 기준금액이 없거나 0 이하: 판단불가
- 실제 이자비용 또는 이자율, 차입금 정보가 부족하면 추가 확인 필요

판정근거에는 다음 내용을 표시한다.

- 계산이자
- 평균차입금
- 적용이자율
- 기간
- 실제 이자비용
- 차이
- 선택한 중요성 프리셋, 기준금액과 허용차이
- 직접 계산 또는 전환권조정 상각 등 보정 내역

## 11. 결과물

엑셀 결과 파일에는 다음 시트를 생성한다.

- 정기보고서목록
- 차입금필터링: 주석 상세 행과 보고기간별 재무상태표 차입 잔액 참고 합계를 함께 표시하되, 참고 합계 행은 검증 계산에서 제외
- 이자비용필터링: 금융비용 주석 상세 행과 오버롤 테스트 적용 기준의 보고기간별 이자비용 참고 합계를 함께 표시하되, 참고 합계 행은 검증 계산에서 제외
- 이자율오버롤테스트
- 추출오류

핵심 검토 시트는 `이자율오버롤테스트`이다.

## 12. 현재까지 반영한 주요 일반화 사항

- 삼성전자처럼 금융수익 및 금융비용 주석에 있는 이자비용을 실제 이자비용으로 사용한다.
- 유한양행처럼 단위가 천원인 금융비용 주석도 백만원으로 변환한다.
- 일진전기처럼 재무제표 주석의 차입금 항목에 있는 이자율 표를 인식한다.
- 에이피알처럼 `재무수익 및 재무비용`이라는 명칭도 금융비용 주석으로 인식한다.
- 애니플러스처럼 차입금 표의 rowspan 때문에 은행명만 남은 행도 이자율/금액 행으로 인식한다.
- `LIBOR+가산금리`는 계약상 대체금리와 조정 조건을 확인할 수 없으면 추가 확인 대상으로 둔다.
- 연결 자료가 있으면 연결 기준을 우선한다.
- 단위 누수로 발생하는 비정상 대형 금액은 원/천원/백만원 기준으로 보정한다.

# DART-OT

**DART-OT**는 DART 공시 API를 기반으로 정기보고서의 차입금 관련 주석을 수집하고, 예상 이자비용과 실제 이자비용의 차이를 검토하는 **이자율 오버롤 테스트 `.xlsx` 파일 생성 도구**입니다.

감사, 검토 업무에서 반복되는 공시 조회, 차입금 내역 필터링, 주석 특이사항 정리와 이자비용 비교를 하나의 데스크톱 GUI에서 처리하는 것을 목표로 합니다. 결과는 회계적 결론이 아니라 추가 검토 대상을 좁히는 참고 자료로 사용합니다.

## 문제 상황

차입금 및 사채의 이자비용 검토는 회사의 재무제표 주석, 정기보고서, 차입 조건, 평균 이자율, 기중 변동 내역 등을 함께 확인해야 하는 작업입니다. 그러나 실무에서는 다음과 같은 비효율이 자주 발생합니다.

- DART에서 여러 연도의 정기보고서를 수동으로 검색하고 다운로드해야 합니다.
- 보고서마다 차입금 주석의 위치와 표현 방식이 달라 필요한 정보를 찾는 데 시간이 많이 걸립니다.
- 차입금, 사채, 유동성 대체 등 관련 내용을 사람이 직접 선별해야 합니다.
- 이자비용이 평균 차입잔액과 공시 이자율에 비추어 적절한지 별도로 계산해야 합니다.
- 여러 회사 또는 여러 기간을 검토할 때 동일한 확인 절차가 반복됩니다.

이러한 수작업 중심의 검토 방식은 시간이 많이 소요되고, 누락, 전사 오류와 검토 기준 불일치가 발생할 가능성이 있습니다.

## 해결 방안

DART-OT는 DART API를 활용해 기본 5개년의 정기보고서를 수집합니다. 사용자는 조회기간을 변경할 수 있습니다. 프로그램은 차입금 관련 공시 내용을 필터링하고, 이자율 오버롤 테스트 결과를 `.xlsx` 파일로 정리합니다.

사용자가 회사명 또는 종목코드와 조회기간을 입력하면 프로그램이 정기보고서 목록 수집, 차입금 주석 추출, 특이사항 정리와 이자비용 비교를 순서대로 처리합니다.

## 주요 기능

- DART 공시 API를 통한 기본 5개년 정기보고서 조회
- 회사명, 보고서명, 종목코드 정리
- 정기보고서 내 차입금, 사채 관련 주석 필터링
- 차입금 주석공시 내용 및 관련 특이사항 요약
- 공시 이자율과 계산된 이자비용을 비교하는 이자율 오버롤 테스트
- 예상 이자비용과 실제 이자비용의 차이를 선택한 중요성 프리셋과 비교
- `_start.bat` 또는 `DART-OT.exe`를 통한 원클릭 실행
- 데스크톱 GUI에서 회사 검색, 회사 선택, `.xlsx` 파일 생성

## `.xlsx` 산출물 구성

산출물은 Office Open XML 형식의 `.xlsx` 파일입니다. 특정 스프레드시트 프로그램을 직접 제어하는 방식이 아니므로 Microsoft Excel 전용 파일은 아니며, `.xlsx` 가져오기를 지원하는 Google Sheets나 한셀 등의 프로그램에서도 열 수 있습니다. 다만 프로그램별 지원 범위에 따라 일부 서식은 다르게 표시될 수 있습니다.

프로그램 실행 결과는 검토 목적에 맞게 다음과 같은 시트로 정리됩니다.

### 1. 정기보고서 목록

선택한 기간의 DART 정기보고서 목록을 표시합니다.

- 회사명
- 보고서명
- 종목코드

### 2. 차입금 필터링 내역

정기보고서에서 차입금 및 사채와 관련된 공시 내용을 별도로 추출해 표시합니다.

- 차입금 관련 주석공시 내용
- 사채 및 차입 조건 관련 정보
- 유동성 대체, 만기, 금리 조건 등 특이사항
- 검토 시 주의가 필요한 문구 또는 이슈

### 3. 이자비용 필터링 내역

정기보고서의 금융원가 주석에서 이자비용 검토에 사용할 수 있는 차입, 금융부채 관련 이자비용 항목을 별도로 추출해 표시합니다.

- 이자비용(금융원가)
- 상각후원가 측정 금융부채 이자비용
- 기타 금융부채 이자비용
- 차입금 이자비용 및 사채 이자비용

### 4. 재무상태표 차입잔액

DART 재무제표 API에서 확인한 차입금 관련 계정과 금액을 별도 시트로 정리합니다. 주석에서 추출한 차입금 금액과 비교할 때 참고할 수 있습니다.

### 5. 이자율 오버롤 테스트

차입금 내역을 기반으로 예상 이자비용과 실제 이자비용의 차이를 확인합니다.

- 차입금 잔액 및 적용 이자율
- 예상 이자비용 계산
- 실제 계상 이자비용과의 차이 분석
- 매출액·세전이익·총자산·자본총계 프리셋 중 선택한 허용차이와 비교

### 6. 추출오류

보고서 수집, 원문 분석과 금액 추출 과정에서 발생한 오류를 회사명, 보고서명, 접수일, 접수번호, 처리 단계와 함께 기록합니다. 오류가 난 보고서를 조용히 제외하지 않고 재검토할 수 있도록 만든 시트입니다.

## 프로젝트 필요성

DART-OT는 단순한 공시 조회 도구가 아니라, 감사, 회계 검토 업무에서 반복되는 차입금 이자율 검토 절차를 일정한 순서로 수행하기 위한 도구입니다.

이 프로젝트를 통해 기대할 수 있는 효과는 다음과 같습니다.

- 반복적인 DART 조회 업무 시간 단축
- 차입금 관련 공시 누락 가능성 감소
- 이자율 오버롤 테스트 기준의 일관성 확보
- 검토 결과를 `.xlsx` 산출물로 보관해 리뷰 및 문서화 용이
- 회사별, 기간별 비교 검토 효율 향상
- GUI 기반 실행으로 비개발자도 쉽게 사용 가능

## 실행 방식

### GitHub Release 배포판

[GitHub Releases](https://github.com/JoonseokWon/DART-OT/releases)에서 `DART-OT-v버전-windows-x64.zip` 파일을 내려받아 압축을 푼 뒤 `DART-OT.exe`를 실행합니다. Release의 실행 파일은 Python 런타임을 포함하므로 사용자 PC에 Python을 별도로 설치할 필요가 없습니다.

배포 ZIP에는 다음 파일이 포함됩니다.

- `DART-OT.exe`: 이자비용 오버롤 테스트
- `DART-Disclosure-Viewer.exe`: DART 공시 원문 조회
- `DART-DEP.exe`: 감가상각비 검토 초안
- `README.md`, `프로세스.md`, `VERSION.txt`

현재 실행 파일에는 코드 서명이 적용되지 않으므로 Windows SmartScreen 경고가 표시될 수 있습니다.

### 소스 저장소에서 실행

최종 애플리케이션은 사용자가 별도의 복잡한 명령어를 입력하지 않아도 실행할 수 있도록 구성합니다.

실행 방식:

```bat
_start.bat
```

또는 Windows 실행 파일을 직접 실행할 수 있습니다.

```bat
DART-OT.exe
```

`_start.bat` 또는 `DART-OT.exe` 실행 시 데스크톱 프로그램 창이 열리고, 사용자는 GUI에서 회사 정보 입력, 회사 검색 및 선택, DART 공시 조회, 차입금 필터링, 이자율 오버롤 테스트 결과 생성 작업을 진행할 수 있습니다.

공시 목록과 원문을 먼저 확인하고 싶은 경우에는 별도 웹 뷰어를 실행할 수 있습니다.

```bat
_disclosure_viewer.bat
```

또는 다음 실행 파일을 직접 실행할 수 있습니다.

```bat
DART-Disclosure-Viewer.exe
```

공시 뷰어는 브라우저에서 회사별 DART 공시 목록을 조회하고, 각 공시의 DART 원문 링크 및 앱 내부 원문 텍스트 검색 기능을 제공합니다.

## 사용 전 준비사항

DART 공시 조회를 위해 OpenDART API 인증키가 필요합니다. 실행 화면에서 API 키와 회사명 또는 종목코드를 입력한 뒤 `회사 검색`을 누르면 후보 회사 목록이 표시됩니다. 목록에서 정확한 회사를 선택한 뒤 파일 생성 버튼을 누르면 정기보고서 수집을 시작합니다.

API 키를 매번 입력하지 않으려면 실행 화면의 `API 키 저장`을 선택하면 됩니다. 저장한 키는 사용자 PC의 로컬 설정 파일(`.dart_ot_config.json`)에 보관됩니다.

## 현재 구현 범위

현재 버전은 DART 재무제표 API와 보고서 원문 XML을 함께 수집합니다. 원문에서는 차입금, 사채, 금융부채, 이자율, 금리, 가중평균, 이자비용과 금융원가 관련 문맥을 찾습니다. 금액과 이자율이 같은 행에 있는 경우에는 금액과 보유기간을 반영해 예상 이자비용을 직접 계산합니다. 총차입금과 이자율만 확인되는 경우에는 평균차입금과 가중평균 이자율을 사용합니다.

실제 이자비용은 금융원가 주석의 상각후원가 측정 금융부채 이자비용, 기타 금융부채 이자비용, 차입금 이자비용과 사채 이자비용 등을 우선 사용합니다. 예상 이자비용과 실제 이자비용의 차이는 실행 화면에서 선택한 중요성 프리셋의 허용차이와 비교합니다.

제공 프리셋은 다음과 같습니다.

- `매출액 1% x 25%`: 기존 PoC 기준이며 기본값
- `매출액 0.5% x 25%`: 매출액 기준 보수적 대안
- `세전이익 5% x 25%`: 이익 중심 기업용 예시
- `총자산 0.5% x 25%`: 자산 중심 기업용 예시
- `자본총계 1% x 25%`: 순자산 중심 기업용 예시

앞의 비율은 전체 중요성 산정 예시이고, 뒤의 `25%`는 이 오버롤 테스트에 적용하는 허용차이 비율입니다. 감사기준은 모든 기업에 적용할 단일 기준이나 백분율을 정하지 않으므로, 프리셋은 감사인의 전문적 판단을 대체하지 않습니다. 선택한 기준금액이 DART에서 검출되지 않거나 0 이하이면 다른 기준으로 자동 대체하지 않고 `판단불가`로 표시합니다. 선택 프리셋, 기준항목, 기준금액, 적용률과 최종 허용차이는 `.xlsx` 파일의 `이자율오버롤테스트` 시트에 기록됩니다.

프리셋의 기준항목과 대표 비율은 [IAASB의 ISA 320 프로젝트 자료](https://www.iaasb.org/consultations-projects/materiality-planning-and-performing-audit-and-evaluation-misstatements-identified-during-audit-isa), [PCAOB의 계획 중요성 브리핑 자료](https://pcaobus.org/News/Events/Documents/02092006_SAGMeeting/Risk_Assessment.pdf), [FRC의 Materiality Thematic Review](https://media.frc.org.uk/documents/Audit_Quality_Thematic_Review_Materiality.pdf)를 참고했습니다. 공개 자료에서도 기준과 비율은 기업과 업무 상황에 따라 달라지므로, 프로그램의 값은 선택 가능한 시작점으로만 제공합니다.

공시자료가 부족하거나 단위와 이자율을 확정하기 어려운 경우에는 원문 확인이 필요합니다. 회사가 별도의 평균 차입이자율을 공시한 경우에는 참고값으로 활용하고, WACC 또는 가중평균자본비용은 차입이자율에서 제외합니다. 금액 열에는 천 단위 쉼표 숫자 서식을 적용합니다.

전기 대비 검출금액합계가 30% 이상 변동하면 `주의요망`으로 표시하고, 별도 사유 열에 주의가 필요한 이유를 설명합니다.

생성된 `.xlsx` 파일은 `outputs` 폴더에 저장되며, 실행 화면에서 결과 파일 또는 저장 폴더를 바로 열 수 있습니다.

## 릴리스 생성 방법

`v`로 시작하는 태그를 GitHub에 푸시하면 `.github/workflows/release.yml`이 Windows 독립 실행 파일을 빌드하고 ZIP 및 SHA-256 체크섬을 GitHub Release에 자동 첨부합니다.

```bash
git tag v0.1.0
git push origin v0.1.0
```

GitHub Actions 화면에서 `Windows release` 워크플로를 수동 실행하면 공개 Release를 만들지 않고 테스트용 빌드 artifact만 생성합니다.

로컬 Windows 환경에서는 Tcl/Tk가 포함된 Python을 준비한 뒤 다음 명령으로 동일한 ZIP을 만들 수 있습니다.

```powershell
python -m pip install -r requirements-build.txt
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build_release.ps1 -Version v0.1.0
```

빌드 스크립트는 GUI 실행에 필요한 `tkinter`가 누락되면 배포를 중단합니다. 생성물은 `release` 폴더에 저장됩니다.

## 별도 감가상각비 테스트 초안

저장소에는 감가상각비와 상각비 주석을 별도로 정리하는 DART-DEP 초안도 포함되어 있습니다. 아직 기대 감가상각비 산정 로직을 완성한 버전은 아니며, DART-OT의 이자비용 검토 기능과는 별도입니다.
